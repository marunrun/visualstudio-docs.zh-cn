# <a name="c6225"></a>C6225
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]
警告 C6225：语义不同的整数类型之间的隐式强制转换：将1或 TRUE 赋给 HRESULT。 请考虑改用 S_FALSE

此警告意味着将使用显式1的值分配或初始化 `HRESULT`。 布尔类型通过非零值指示成功;`HRESULT` 中的成功（`S_OK`）表示值0。 此警告通常是由于意外混淆了布尔值和 `HRESULT` 类型引起的。 若要指示成功，应使用符号常量 `S_OK`。

## <a name="example"></a>示例

在下面的代码中，分配 `HRESULT` 将生成此警告：

```cpp
#include <windows.h>

VOID f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  if (SUCCEEDED(CoGetMalloc(1, &pMalloc)))
  {
    // code ...
    hr = S_OK;
  }
  else
  {
    // code ...
    hr = 1;
  }
}
```

若要更正此警告，请使用以下代码：

```cpp
VOID f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  if (SUCCEEDED(CoGetMalloc(1, &pMalloc)))
  {
    hr = S_OK;
    // code ...
  }
  else
  {
    hr = S_FALSE;
    // code ...
  }
}
```

对于此警告，`SCODE` 类型等效于 `HRESULT`。

若要指示失败，应使用 `E_FAIL`或另一个常数。
